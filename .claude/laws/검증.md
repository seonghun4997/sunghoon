# 검증 — 완료 보고·검증 계정·스모크 필독

## 완료 선언 기준
- 실행 로그(빌드 통과·HTTP 200·DB 행 반환 등)가 있을 때만 ✅.
- 없으면 "미검증"이라 명시한다. "됐다"는 로그가 증명할 때만.

## 검증 다운그레이드 금지
- 막혔을 때 검증 범위를 줄여서 "완료"로 보고하는 것을 금지한다 (예: "화면은 못 보니 쿼리만 검증").
- 축소 검증은 "부분 검증"이며, 남은 미검증 항목을 명시해야 한다.
- 순서 고정: ①막힌 원인을 한 문장으로 확정 → ②스스로 치울 수 있으면 치우기 → ③못 치우는 것만 콕 집어 요청.
- 스스로 치울 수 있는 것: SESSION_SECRET 등 랜덤 시크릿(`openssl rand`로 생성), 테스트 계정 시드, 로컬 설정 파일, 캐시 삭제.
- 오너에게 요청해야 하는 것: 외부에서 발급받는 값(결제·외부 API 키)뿐. 요청 시 "어느 서비스 → 어느 화면 → 어느 값"까지 콕 집고, 코드 근거를 함께 제시한다.
- 어느 경우든 시크릿 값 자체는 채팅·커밋·문서에 노출 금지.

## 검증 계정 (test_검증봇)
- "로그인 벽이라 미검증"은 완료 보고 사유로 인정하지 않는다.
- 검증 계정은 `test_검증봇` 하나로 통일한다. 새 테스트 계정을 만들지 않는다.
- 역할 전환이 필요하면 동일 계정의 권한을 임시 조정 후 원복한다.
- 자격증명은 채팅·커밋·문서·로그 어디에도 노출 금지. 비밀번호는 `.env.test`에만 저장하며 gitignore 확인을 로그로 증명한다.
- 오너의 실계정·실비밀번호는 어떤 경우에도 요청·사용·저장하지 않는다.
- 테스트 계정의 쓰기 행동이 실데이터·실지표를 오염시키지 않도록 표시·격리한다.
- 같은 계정을 배포 후 스모크 검증에도 사용한다.
