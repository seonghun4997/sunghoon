# 병렬 작업 — git 쓰기·같은 repo 다중 창 필독

## 창 전담제
- 각 창(세션)은 시작할 때 자기 담당 구역을 선언한다 (예: "이 창은 마케팅 허브 전담").
- 담당 밖 파일은 읽기만 하고 수정하지 않는다.
- 착수 전 BACKLOG.md에 자기 항목을 "⏸작업중 — [구역명] [날짜]"로 표시한다.
- 다른 창이 이미 ⏸작업중으로 잡은 구역·파일은 건드리지 않는다 (대장이 잠금판).

## 공용 파일 주의
- 공용 파일(lib/constants.js, CLAUDE.md, BACKLOG.md, package.json, 전역 스타일)은 충돌 1순위다.
- 수정이 필요하면 최소 범위로, 커밋 직전에 몰아서, pull 직후에 한다.

## 커밋 절차 고정
1. `git pull` 먼저
2. 충돌 나면 임의 병합 금지 — 겹친 파일 목록을 오너에게 보고하고 판단받기
3. 자기 구역 파일만 스테이징 (`git add .` 금지, 파일 지정)
4. 푸시

## 락 래퍼 의무
- 빌드·git 쓰기(add/commit/push)·dev 서버 기동은 반드시 `node scripts/withlock.js -- <명령>`로 실행한다. 맨몸 실행 금지.
- 락 대기 메시지(⏸)가 뜨면 정상 — 조용히 기다린다. 10분 초과(⛔)만 오너에게 보고.
- 각 창은 세션 시작 시 `CLAUDE_WINDOW` 환경변수에 자기 이름(예: 디자인허브·편집허브)을 넣는다 — 락에 "누가" 찍히게.
- `.claude/repo.lock`은 gitignore에 추가 (커밋 금지).

## DB·마이그레이션
- SQL은 병렬 금지: 한 번에 한 창만, 대장에 선언 후 실행.

## 낯선 수정 발견 시
- 다른 창의 미커밋 변경을 발견하면 덮어쓰지 말고 멈추고 보고한다.
- 세션을 끝내거나 전환할 때: 자기 ⏸ 항목에 진행 상황 한 줄을 남긴다 (다음 창이 이어받을 수 있게).
